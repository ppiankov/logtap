# logtap Work Orders

## Phase 1: Receiver (Week 1)

### WO-01: TCP/HTTP receiver with Loki push API

**Goal**: Accept log streams via `POST /loki/api/v1/push` and write to disk.

**CLI interface**:
```
logtap recv --listen :9000 --dir ./capture --max-disk 50GB
```

**Flags**:
| Flag | Default | Description |
|------|---------|-------------|
| `--listen` | `:9000` | Address to bind |
| `--dir` | `./capture` | Output directory |
| `--max-disk` | `50GB` | Hard cap on total disk usage |
| `--max-file` | `1GB` | Rotate file at this size |
| `--compress` | `true` | Compress rotated files with zstd |

**Behavior**:
- Accept `POST /loki/api/v1/push` (Loki push API format)
- Also accept raw newline-delimited JSON on `POST /logtap/raw`
- Parse Loki push payload → extract log lines with labels
- Write to JSONL: `{"ts":"...","labels":{...},"msg":"..."}`
- Rotate files at `--max-file` size
- Delete oldest files when `--max-disk` exceeded
- Never block sender — return 200 immediately, drop if buffer full
- Expose `/metrics` for own health (prometheus format)

**Internal metrics**:
- `logtap_logs_received_total`
- `logtap_logs_dropped_total`
- `logtap_bytes_written_total`
- `logtap_disk_usage_bytes`
- `logtap_active_connections`
- `logtap_backpressure_events_total`

**Files**:
- `cmd/logtap/main.go` — cobra root + recv subcommand
- `internal/recv/server.go` — HTTP server, Loki push handler
- `internal/recv/writer.go` — buffered JSONL writer
- `internal/rotate/rotator.go` — file rotation + disk cap enforcement
- `internal/rotate/rotator_test.go`

**Verification**:
```bash
# Start receiver
logtap recv --listen :9000 --dir /tmp/capture --max-disk 1GB

# Send test data
curl -X POST http://localhost:9000/loki/api/v1/push \
  -H 'Content-Type: application/json' \
  -d '{"streams":[{"stream":{"app":"test"},"values":[["1234567890000000000","hello world"]]}]}'

# Verify written
cat /tmp/capture/*.jsonl | jq .

# Stress test
go test -run TestBackpressure -race ./internal/recv/
```

### WO-02: Minimal TUI

**Goal**: Show live stats while receiving.

**Display**:
```
logtap v0.1.0 | :9000 | ./capture

Connections:  3
Logs/sec:     48,232
Bytes/sec:    62 MB/s
Disk used:    14.2 GB / 50.0 GB
Dropped:      0

Top talkers:
  api-service       21,412/s
  worker-service    14,087/s
  gateway            9,201/s
```

**Behavior**:
- Update every 1 second
- Top talkers by label (namespace/pod/app)
- Show drop counter prominently (red if > 0)
- Exit cleanly on Ctrl+C (flush buffers, close files)

**Dependencies**: `github.com/charmbracelet/bubbletea` (optional) or raw terminal output.

**Files**:
- `internal/recv/tui.go` — terminal rendering
- `internal/recv/stats.go` — atomic counters for live stats

**Verification**: Visual — run recv with a sender and confirm display updates.

## Phase 2: Cluster Integration (Week 2)

### WO-03: Agent detection and config patching

**Goal**: `logtap install` detects logging agent and patches config.

**CLI interface**:
```
logtap install [--agent alloy|vector|fluentbit] [--target logtap-host:9000]
logtap install --dry-run
```

**Behavior**:
- Auto-detect agent: `kubectl get ds -A` looking for alloy/vector/fluent
- If ambiguous, require `--agent` flag
- Read current configmap
- Inject mirror output block wrapped in marker comments:
  ```
  # --- logtap start ---
  # generated by logtap — do not edit manually
  ...
  # --- logtap end ---
  ```
- Detect GitOps management (ArgoCD/Flux annotations) → warn, don't proceed unless `--force`
- Show diff of what will change (`--dry-run` default)
- Restart daemonset with rolling update

**Supported agents (v1)**:
- Grafana Alloy: inject `loki.write "logtap"` block with annotation filter
- Vector: inject `[sinks.logtap]` TCP socket sink with filter transform

**Files**:
- `internal/k8s/detect.go` — agent auto-detection
- `internal/k8s/gitops.go` — ArgoCD/Flux detection + warning
- `internal/patch/alloy.go` — Alloy config patch generation
- `internal/patch/vector.go` — Vector config patch generation
- `internal/patch/apply.go` — configmap read/write/diff
- `cmd/logtap/install.go` — install subcommand

**Verification**:
```bash
logtap install --dry-run
logtap install --target 10.0.0.5:9000
kubectl get cm -n alloy alloy-config -o yaml | grep logtap
```

### WO-04: Annotation-based enable/disable

**Goal**: `logtap enable` / `logtap disable` toggles mirroring per workload.

**CLI interface**:
```
logtap enable --namespace myservice
logtap enable --selector app=payments
logtap enable --deployment api-gateway
logtap disable --namespace myservice
```

**Behavior**:
- Add annotation `logtap.dev/enabled: "true"` to matching resources
- Support: namespace, deployment, statefulset, daemonset, selector
- Show what will be annotated before applying
- `disable` removes annotation
- No redeploy needed — annotation is metadata-only

**Files**:
- `cmd/logtap/enable.go`
- `cmd/logtap/disable.go`
- `internal/k8s/annotate.go`

**Verification**:
```bash
logtap enable --selector app=payments
kubectl get deploy payments -o jsonpath='{.metadata.annotations}'
logtap disable --selector app=payments
```

### WO-05: Clean uninstall

**Goal**: `logtap uninstall` removes all logtap artifacts from cluster.

**CLI interface**:
```
logtap uninstall [--dry-run]
```

**Behavior**:
- Find marker blocks in logging agent configmap
- Remove only content between `# --- logtap start ---` and `# --- logtap end ---`
- Remove logtap annotations from all resources
- Restart daemonset
- Show diff before applying

**Files**:
- `cmd/logtap/uninstall.go`
- `internal/patch/remove.go`

**Verification**:
```bash
logtap uninstall --dry-run
logtap uninstall
kubectl get cm -n alloy alloy-config -o yaml | grep -c logtap  # should be 0
```

## Phase 3: Hardening (Week 3)

### WO-06: Backpressure and stress testing

**Goal**: Verify receiver handles 100+ MB/s without blocking senders.

**Tests**:
- 100 MB/s sustained write for 5 minutes
- Disk full scenario (verify oldest files deleted, counter incremented)
- Sender faster than writer (verify drop counter, never blocks)
- Connection storm (100 concurrent connections)
- Graceful shutdown (Ctrl+C flushes all buffers)

**Files**:
- `internal/recv/stress_test.go`
- `internal/rotate/rotate_test.go`

---

### WO-07: Security guardrails

**Goal**: Prevent PII/PCI leaks by default.

**Behavior**:
- Default: refuse connections from prod-labeled namespaces
- Flag `--allow-prod` overrides (with warning banner in TUI)
- Optional `--redact` flag with regex patterns for PII
- TLS support via `--tls-cert` and `--tls-key`
- All traffic logged to audit file (connection source, bytes, duration)

**Files**:
- `internal/recv/security.go`
- `internal/recv/redact.go`
